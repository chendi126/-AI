# 傻毛宠物
> 一个面向养宠人和准养宠人的宠物生活百科 App。  
> 最终形态不是“宠物版小红书”，而是“宠物问题解决平台”。

## 这个产品最终会长成什么样
傻毛宠物最终会是一个把内容、知识、AI 和服务串起来的移动端 App。

用户进入产品，不是为了单纯逛内容，而是为了尽快回答几个问题：
- 这是什么
- 严不严重
- 该怎么做
- 去哪里做
- 买什么，找谁，怎么选

产品会同时服务四类人：
- 已经养宠、但经常遇到问题的人
- 刚开始养宠的新手
- 还没养宠、但正在做决策的人
- 想分享经验、建立信任和归属感的资深养宠人

## 最终产品结构

| 层级 | 作用 | 最终结果 |
|---|---|---|
| 内容层 | 像小红书一样，承载真实经验、故事和日常分享 | 用户愿意留下来，看见“和我一样的人” |
| 知识层 | 结构化沉淀品种百科、喂养、健康、护理、训练、领养、后事 | 用户能快速找到可靠答案 |
| AI 层 | 把问题翻译成可执行的下一步，而不是只给泛泛而谈的回复 | 用户知道先做什么、再做什么 |
| 服务层 | 接入医院、训练、保险、用品、领养等真实服务 | 用户能直接完成选择和行动 |
| 情感层 | 记录宠物、纪念离世、连接同类人群 | 产品不只是工具，也是一种陪伴 |

## 典型使用路径

### 新手养宠
1. 先做宠物档案和新手问卷。
2. 自动生成品种/年龄/阶段清单。
3. 需要时随时问 AI。
4. 再去看对应品种、喂养和训练内容。

### 遇到异常症状
1. 用户拍照、描述或语音输入。
2. AI 先做分诊和风险提示。
3. 给出可执行建议。
4. 必要时推荐医院、保险或进一步咨询。

### 选粮、选用品、选服务
1. 先看真实经验和品种适配建议。
2. 再做对比工具。
3. 最后再跳转到商品或本地服务。

### 领养和纪念
1. 领养入口与流浪动物信息。
2. 捐赠、救助和本地资源。
3. 宠物离世后的纪念空间。

## 产品边界

这个产品不是：
- 通用 AI 聊天机器人
- 纯内容社区
- 硬件优先的平台
- 直接替代兽医的诊断工具

它更像是一个“宠物问题解决系统”。

## 商业方向

早期更适合的收入顺序是：
1. 单次问答和短周期陪跑
2. 会员订阅
3. 服务撮合和分成
4. B 端合作和白标能力

不建议一开始就重广告、重商城、重硬件。

## 当前项目状态

当前仓库里的是一个移动端 Web 原型，入口在 `index.html`。
现阶段主要验证：
- 用户是否真的会来问问题
- 哪些问题最值钱
- 用户是否愿意付费
- 哪个渠道最容易获客
