# 傻毛宠物 - 产品需求文档（PRD）

## 一、文档概述

### 1.1 文档目的

本文档定义「傻毛宠物」App 的产品定位、目标用户、功能架构、业务流程及迭代规划，作为技术、设计、运营团队的统一执行依据。

### 1.2 产品定位

**「傻毛宠物」是一个以宠物品种知识库为核心的垂直社交 + 电商 App。**

- **一句话定位**："最懂你家宠物是什么品种的养宠百科全书"
- **核心价值主张**：用结构化品种知识解决养宠信息不对称问题，用社区连接养宠人，用精准推荐完成消费闭环
- **差异化关键词**：品种垂直、知识结构化、AI 问答、按品种推荐商品

### 1.3 目标市场

| 维度        | 描述                                          |
| --------- | ------------------------------------------- |
| **市场规模**  | 2024 年中国宠物市场规模约 3000 亿元，宠物主超 7500 万         |
| **目标用户池** | 25-40 岁城市养宠人群，约 4000-5000 万                 |
| **渗透率目标** | 第一年 50 万注册用户，DAU 3 万；第二年 200 万注册用户，DAU 12 万 |

## 四、核心功能详细需求

### 4.1 品种百科（核心模块）

#### 4.1.1 品种列表页

**需求描述**：用户可浏览所有收录的宠物品种，支持多维度筛选和搜索。

**功能细节**：

- 按大类筛选：犬类、猫类、异宠（小宠、鸟类、爬宠、水族）
- 二级筛选（犬类）：按体型（大型/中型/小型/超小型）、按毛发（长毛/短毛/卷毛/无毛）、按功能（工作犬/伴侣犬/猎犬）
- 搜索：支持品种名、别名、关键词搜索（如"不掉毛"）
- 列表展示：品种封面图、中文名、英文名、体型标签、性格关键词
- 快捷入口：热门品种、最近浏览、品种匹配测试

**交互说明**：

- 下拉刷新，上拉加载更多
- 点击品种卡片进入品种详情页
- 支持左右滑动快速切换大类

#### 4.1.2 品种详情页

**需求描述**：每个品种的核心信息展示页，是产品的核心内容资产。

**页面结构**（从上到下）：

1. **品种封面区**
   
   - 品种高清图片（轮播，3-5 张）
   - 品种中文名 + 英文名
   - 收藏按钮、分享按钮
   - 一句话简介（如："温顺友善的家庭伴侣犬，对小孩极其耐心"）

2. **基础信息卡**
   
   - 体型：大型犬
   - 寿命：10-12 年
   - 原产地：英国
   - 性格标签：温顺、友善、聪明、活跃
   - 饲养难度：⭐⭐⭐（3/5）
   - 适合人群：有庭院家庭、有小孩家庭、运动爱好者
   - 饲养成本：💰💰💰（3/5）

3. **Tab 内容区**（四个 Tab）
   **Tab 1: 饮食指南**
   
   - 推荐食物类型（幼犬期/成犬期/老年期）
   - 每日食量参考表（按体重）
   - 禁忌食物清单（带毒性说明）
   - 特殊饮食需求（如易过敏需低敏粮）
   - 推荐商品卡片（关联商城）
   
   **Tab 2: 健康档案**
   
   - 常见遗传病（髋关节发育不良、眼部疾病等）
   - 高发疾病及预防
   - 疫苗接种时间表
   - 建议体检项目及频率
   - 紧急症状识别（红色警报）
   - ⚠️ 免责声明："本内容仅供参考，具体医疗建议请咨询执业兽医"
   
   **Tab 3: 护理指南**
   
   - 毛发护理频率及工具
   - 运动量需求（每天 xx 分钟）
   - 训练难度及建议
   - 适合的气候环境
   - 居家布置建议
   
   **Tab 4: 社区内容**
   
   - 该品种下的热门帖子
   - 同品种用户交流区
   - "我有 XX（品种）" 用户入口

4. **底部操作区**
   
   - "添加我的 XX" 按钮（引导用户添加宠物档案）
   - "查看相关商品" 按钮

**数据要求**：每个品种字段完整度需达到 90% 以上才能上线。

#### 4.1.3 品种对比工具

**需求描述**：支持同时对比 2-3 个品种的关键指标，帮助用户选择。

**对比维度**：

- 体型、寿命、性格、饲养难度、运动量、掉毛程度、训练难度、与儿童友好度、与其他宠物友好度、适合公寓/庭院、饲养成本

**展示形式**：左右并列对比表格，差异项高亮显示。

#### 4.1.4 品种匹配测试

**需求描述**：帮助潜在养宠人找到适合自己的宠物品种。

**测试流程**：

1. 选择大类（犬/猫/异宠）
2. 回答 8-10 道问题：
   - 居住空间大小
   - 是否有小孩/老人
   - 每天能陪伴的时间
   - 对掉毛的接受度
   - 运动能力
   - 预算范围
   - 是否有养宠经验
3. 生成推荐结果（Top 3 品种）
4. 每个推荐附匹配度百分比和推荐理由
5. 引导查看品种详情或保存结果

### 4.2 内容社区

#### 4.2.1 首页 Feed

**需求描述**：双列瀑布流展示社区内容，类似小红书。

**内容类型**：

- 图文帖子（最多 9 张图）
- 短视频（最长 60 秒）
- 图文+视频混合

**推荐策略**（逐步迭代）：

- **V1**：按时间倒序 + 运营精选置顶
- **V2**：基于品种标签的协同过滤（喜欢金毛的也看拉布拉多）
- **V3**：个性化推荐算法 + 兴趣标签 + 用户行为

**卡片信息**：

- 封面图/视频缩略图
- 标题（最多 2 行，超出省略）
- 作者头像 + 昵称
- 点赞数、品种标签（如 #金毛）
- 认证标识（专家、达人）

**筛选器**：

- 全部 / 关注
- 按品种筛选（用户已添加的宠物品种优先）
- 按内容类型：经验分享 / 日常晒宠 / 问题求助 / 商品评测

#### 4.2.2 发布器

**需求描述**：用户发布内容的多媒体编辑器。

**功能**：

- 图片上传（最多 9 张，支持裁剪、滤镜、添加品种贴纸）
- 视频上传（最长 60 秒，支持基础剪辑、添加字幕）
- 文本编辑（最多 2000 字，支持 @用户、#话题、#品种标签）
- 品种标签：必须选择至少一个关联品种（最多 3 个）
- 话题标签：自动推荐热门话题
- 发布位置：可选（宠物医院、宠物店等 POI）
- 可见范围：公开 / 仅粉丝 / 私密

**AI 辅助功能**：

- 智能标题推荐（基于图片内容识别）
- 违禁词检测（发布前自动提醒）
- 品种自动识别（上传宠物照片时自动推荐品种标签）

#### 4.2.3 内容详情页

**页面结构**：

- 作者信息区（头像、昵称、关注按钮）
- 内容区（图片/视频轮播 + 文字）
- 互动区（点赞、收藏、评论、分享）
- 评论区（支持评论回复、点赞评论、@用户）
- 相关推荐（同品种的其他热门帖子）

#### 4.2.4 消息系统

**消息类型**：

- 互动通知：点赞、评论、关注、@提及
- 系统通知：审核结果、活动通知、等级升级
- 私信：支持文字、图片、表情
- 专家咨询：用户向认证专家发起提问（付费/免费）

### 4.3 宠物商城

#### 4.3.1 商品推荐页（V1：联盟电商）

**需求描述**：基于用户浏览和宠物品种，推荐精准商品。

**页面结构**：

- 顶部 Banner：活动/优惠券
- 品种快捷入口："为我的金毛推荐""为我的布偶猫推荐"
- 商品分类：主粮、零食、保健品、玩具、用品、医疗
- 商品列表：双列瀑布流
- 商品卡片：图片、名称、价格、销量、评价数、"适合 XX 品种"标签

**推荐逻辑**：

- 无宠物档案：展示热门通用商品
- 有宠物档案：优先展示适合该品种/年龄段的商品
- 基于浏览历史：相似商品推荐

#### 4.3.2 商品详情页

**页面结构**：

- 商品图片轮播
- 商品名称 + 价格 + 销量
- 适用品种标签
- 商品规格选择（重量/口味/尺寸）
- 商品详情（图文）
- 用户评价（可按品种筛选评价）
- 相关推荐
- 底部操作：加入购物车 / 立即购买（跳转至淘宝/京东联盟链接）

**V2 自营商城升级**：

- 增加自营标识
- 优惠券系统
- 满减活动
- 会员价
- 购物车、订单管理、物流追踪

### 4.4 傻毛AI

#### 4.4.1 功能定位

- **不是**：宠物医生（不诊断、不开药）
- **是**：宠物知识问答助手、养宠问题查询工具、商品导购顾问

#### 4.4.2 交互设计

**入口**：

- 首页底部 Tab 中央悬浮按钮（类似微信"发现"）
- 品种详情页"问傻毛"按钮
- 全局搜索框旁语音按钮

**对话界面**：

- 顶部：傻毛形象（卡通宠物头像）+ "你好，我是傻毛，你的养宠小助手"
- 中部：对话气泡区
- 底部：输入框 + 语音按钮 + 快捷问题卡片

**快捷问题（动态推荐）**：

- 根据用户当前场景推荐：
  - 在品种详情页："金毛幼犬一天吃多少？""金毛不能吃什么？"
  - 在首页："新手养狗需要注意什么？""推荐适合公寓养的狗"
  - 在商城："2 个月幼犬推荐什么狗粮？"

#### 4.4.3 回答规范

**必须包含的元素**：

- 清晰、简洁的回答正文
- 信息来源标注（如"根据品种百科-金毛-饮食指南"）
- 相关推荐（如相关文章、相关商品）
- 医疗相关回答必须附带："⚠️ 本回答仅供参考，不能替代专业兽医诊疗。如宠物症状严重，请立即就医。"

**回答限制**：

- 拒绝回答：具体用药剂量、疾病诊断、治疗方案
- 遇到医疗问题：提供知识科普 + 强烈建议就医 + 推荐附近宠物医院

#### 4.4.4 技术架构（RAG）

```
用户提问
    ↓
意图识别（品种查询/疾病咨询/商品推荐/日常闲聊）
    ↓
向量化检索（品种知识库/UGC内容/商品库）
    ↓
大模型生成回答（GPT-4/Claude/国内大模型API）
    ↓
后处理（安全过滤、来源标注、格式美化）
    ↓
输出给用户
```

### 4.5 用户系统

#### 4.5.1 注册登录

**方式**：手机号 + 验证码（微信/Apple 登录可选）

**新用户引导**：

1. 欢迎页：产品价值展示（3 张引导页）
2. 选择身份：已有宠物 / 准备养宠 / 纯云吸宠
3. 已有宠物：添加宠物档案（名称、品种、年龄、性别、体重、头像）
4. 准备养宠：进入品种匹配测试
5. 纯云吸宠：直接浏览社区

#### 4.5.2 宠物档案

**支持多宠物**：一个用户可添加多只宠物，每只宠物独立档案。

**档案字段**：

- 基本信息：昵称、头像、品种、性别、出生日期/年龄、体重、是否绝育
- 健康记录：疫苗记录、驱虫记录、体检记录、病史记录（可上传图片）
- 饮食偏好：当前主粮品牌、零食偏好、过敏食物
- 设置提醒：下次疫苗时间、下次驱虫时间、生日提醒

**档案价值**：

- 个性化推荐：基于品种+年龄+体重推荐内容
- 智能提醒：到期自动推送通知
- 社区标签：发帖时自动带上宠物档案

## 八、交互原型说明

### 8.1 核心页面流程

**用户首次使用流程**：

```
启动 App → 欢迎页（3张）→ 选择身份（已有宠物/准备养宠/云吸宠）
  ├─ 已有宠物 → 添加宠物档案（名称/品种/年龄/头像）→ 首页
  ├─ 准备养宠 → 品种匹配测试 → 推荐结果 → 品种详情 → 首页
  └─ 云吸宠 → 直接首页
```

**核心浏览流程**：

```
首页 Feed → 品种标签 → 品种详情 → 饮食/健康/护理/社区 Tab
  ↓
傻毛AI 问答 ← 商品推荐 ← 相关帖子
```

**内容发布流程**：

```
点击发布按钮 → 选择图片/视频 → 编辑文字 → 选择品种标签 → AI 辅助检测 → 发布 → 审核 → 上线
```
